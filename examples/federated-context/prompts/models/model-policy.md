# Model policy (fictional example)

Organization guidance for model and thinking-level selection. Kept relative on purpose — model names and prices drift.

- Default to the organization's approved mid-tier model for routine implementation and review work.
- Escalate to the strongest approved model (and a higher thinking level) when a wrong answer is expensive: money paths, migrations, incident diagnosis, irreversible operations.
- Use the cheapest approved model for high-volume, low-risk work: summarization, triage, formatting, boilerplate.
- If a task involves regulated data (payments, PII), only models on the approved list may be used — no exceptions for speed.
