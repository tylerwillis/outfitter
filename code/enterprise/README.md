# Outfitter Enterprise Code

Everything in this directory and its descendants is licensed under the [Unsupervised Enterprise License](./LICENSE), a Business Source-style license: you may read, modify, and test this code freely, but production use requires an enterprise agreement. Everything outside `code/enterprise/**` — including the CLI in `code/cli` — is MIT-licensed under the root [`LICENSE.md`](../../LICENSE.md). Individual and team use of Outfitter is free and stays free; this directory exists so org-scale capabilities have a clear commercial boundary from day one instead of a license change later. Outfitter's monetization is enterprise licenses for these features plus paid training and support.

Today this directory gates private GitHub profile catalog support: `privateCatalog.js` defines the package-visible commercial boundary, shared policy lives in `shared/`, CLI sync/settings helpers live in `cli/`, and Pi-native onboarding helpers live in `pi-extension/`. Package staging imports the policy marker and emits `private-catalog-boundary.json` so the published package carries executed enterprise policy artifacts, while public sync/setup behavior remains non-enforcing and relies on the user's ambient Git configuration. Future org-scale governance and audit tooling — policy enforcement (`enforce` mode), audit collectors, retention, and organization reporting, per the [org-catalog governance design spec](../../docs/specs/org-catalog-governance.md) — will also live here; the underlying mechanisms and schemas remain MIT.
